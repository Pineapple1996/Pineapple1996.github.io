发表于 2017-11-30 | 分类于 学习笔记 | 阅读次数

什么是git？

Git是一个开源的分布式版本控制系统，用以有效、高速的处理从很小到非常大的项目版本管理。

简单来讲，就是你保存管理代码的地方。
你可以把git想象成一个仓库，而你是一名雕刻师，你要完成一个产品，需要经历一个开发过程。你可能每天完成作品一部分，当天下班的时候，你就把你还未完成的作品放在仓库里，第二天继续。这样不断循环，直到你完成这个作品为止。这个过程中还允许你的伙伴来提出他们的改进意见，帮助你完成你的作品，当然你也可以不接受他们的意见。

准备工作

在正式开始之前，你还需要确认你的电脑是否安装了git。如果还没安装你可以点击这里，去看看怎样安装。

创建版本库


我们先来理解下Git 工作区、暂存区和版本库概念

▶工作区：就是你在电脑里能看到的目录。
▶暂存区：英文叫stage, 或index。一般存放在 “.git目录下” 下的index文件（.git/index）中，所以我们把暂存区有时也叫作索引（index）。
▶版本库：工作区有一个隐藏目录.git，这个不算工作区，而是Git的版本库。

创建一个版本库非常简单，首先，选择一个合适的地方，创建一个空目录。

git diff的一些用法

●尚未缓存的改动：git diff
●查看已缓存的改动： git diff –cached
●查看已缓存的与未缓存的所有改动：git diff HEAD
●显示摘要而非整个 diff：git diff –stat

接下来,我们来看命令 git checkout –readme.txt 意思，就是把readme.txt文件在工作区做的修改全部撤销，这里有2种情况，如下：

1、readme.txt自动修改后，还没有放到暂存区，使用 撤销修改就回到和版本库一模一样的状态。
2、另外一种是readme.txt已经放入暂存区了，接着又作了修改，撤销修改就回到添加暂存区后的状态。
注意：命令git checkout — readme.txt 中的 — 很重要，如果没有 — 的话，那么命令变成创建分支了。
